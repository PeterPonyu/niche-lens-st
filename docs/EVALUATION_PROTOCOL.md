# NicheLens-ST evaluation protocol

Status: **PRE-REGISTRATION** — written before real results exist; amendments must be dated and justified.

> **Admissibility rule.** The single-section MERFISH fallback run (`dataset_card_id: niche_merfish_slice`,
> `fallback_warning` set in `results/paper_metrics/leaderboard.json`) is **not** admissible evidence for
> any claim in this protocol. It is a downsized smoke-test: the conserved/sample-specific distinction is

### C1 — Reproducible niche prototypes

**Hypothesis.** Contrastive spatial-neighborhood embeddings assign cells to niche prototypes that are
reproducible across tissue sections at the prototype-assignment level (invariant to arbitrary cluster
numbering).

#### Primary evidence: synthetic benchmark

| Metric | Implementation | What it locks |
|---|---|---|
| ARI | `adjusted_rand(pred_id, true_id)` | Pair-counting agreement, label-permutation invariant |
| AMI | `adjusted_mutual_info(pred_id, true_id)` | Information-theoretic agreement, chance-corrected |
| NMI | `normalized_mutual_info(pred_id, true_id)` | Absolute information overlap (reported alongside ARI) |
| Homogeneity | `homogeneity(pred_id, true_id)` | Each cluster contains members of a single true niche |
| Completeness | `completeness(pred_id, true_id)` | All members of a true niche land in the same cluster |
| V-measure | `v_measure(pred_id, true_id)` | Harmonic mean of homogeneity and completeness |
| Neighbor label agreement | `neighbor_label_agreement(pred_id, edges)` | Spatial coherence of recovered niches over the kNN graph |
| Macro-F1 | `macro_f1(pred_id, true_id)` | Hungarian-aligned per-class F1 averaged unweighted |
| Marker recall@k | `marker_recall_at_k(pred_markers, true_markers, k=5)` | Per-prototype marker recovery after Hungarian alignment |

**Comparators.** NicheLens-ST vs. all of:
- Neighborhood baseline (clean-room; spatial context only, no contrastive objective)
- PCA baseline (clean-room; linear dimensionality reduction, no spatial context)
- Diffusion baseline (clean-room; graph diffusion embedding)
- Random labeling (lower bound; uniform random prototype assignment)

**Datasets (synthetic).** All synthetic instances with `n_sections >= 2` generated by
`nichelens_st.synth.generator` (parameters per `docs/SYNTHETIC_BENCHMARK.md`; ground-truth
`prototype_id` from `truth/prototype_id.npy`, `proto_kind` from `truth/proto_kind.json`).

**Decision rule.** *Direction:* NicheLens-ST ARI, AMI, and `neighbor_label_agreement` must each exceed
every clean-room baseline AND random labeling on the synthetic benchmark.
**Numeric pass-threshold: to be fixed by pre-hoc synthetic calibration before touching real data.**
Calibration procedure: set threshold at the 95th percentile of random-labeling scores over ≥ 100
synthetic instances drawn from the full generator-parameter grid; a method must exceed this threshold
on ≥ 80 % of instances.

#### Secondary evidence: real-data domain recovery

| Metric | Role |
|---|---|
| ARI, AMI, Macro-F1 | Prototype assignment vs. domain annotation (where available in data card) |
| Neighbor label agreement | Spatial coherence over tissue kNN graph |

**Datasets (real, multi-section required).** From `data/cards/`:

| Card ID | Reason for inclusion |
|---|---|
| `abc_atlas_zhuang_merfish` | Atlas-scale MERFISH; multiple sections available |
| `merfish_hypothalamus_moffitt` | MERFISH; known cell-type annotations; multiple subjects |
| `wang2025_matched_tma` | Matched TMA; multi-sample by design |
| `xenium_lymph_node` | Xenium; multi-section capable |

Eligibility criterion: data card must document ≥ 2 tissue sections from the same tissue type.

**Decision rule.** *Direction:* NicheLens-ST ARI/AMI vs. domain annotation exceeds every clean-room
baseline AND random labeling on eligible real datasets.
**Numeric threshold: to be fixed by pre-hoc synthetic calibration before touching real data**
(threshold transferred from the synthetic grid calibration above).

**Amendment 2026-06-06 (#288 — supervised niche-recovery table; single-dataset scope).**
A secondary, caveated supervised niche-recovery table is now implemented in
`scripts/run_real_niche.py` (`--supervised-metrics` → `outputs/supervised_metrics.json`): for each
available author reference it reports ARI/NMI/AMI/macro-F1 of (a) the learned prototype assignment and
(b) k-means on the embedding at `k = n_ref_classes` over 3 seeds (run-to-run distribution). *Scope is
single-dataset for now:* only `codex_spleen_goltsev2018` ships author dual ground truth
(`obs['niche']`, `obs['cell_type']`), and **both labels are clustering-derived, not hand-annotated**.
Per the intrinsic-metrics-first / GT-skeptic rule these scores are **secondary** — written to a
separate file, never folded into the headline intrinsic `metrics.json`, and not admissible as a
standalone accuracy claim. The original #288 multi-dataset, rank-aggregation panel is **deferred**
until a second niche-GT dataset lands (HuBMAP is access-blocked). The intrinsic metrics (silhouette,
neighbor-label-agreement, per-niche Moran's I) remain the primary yardstick.

### C2 — Conserved and sample-specific microenvironments

**Hypothesis.** Niche prototypes learned by NicheLens-ST recover the conserved/sample-specific
structure of cellular microenvironments across multiple samples: conserved prototypes appear in every
section and sample-specific prototypes appear in a strict subset.

> **Required condition.** This claim requires ≥ 2 tissue sections as input.
> Single-section results are inadmissible (see admissibility rule above).

#### Metrics

| Metric | Implementation | What it locks |
|---|---|---|
| Proto-kind accuracy | `proto_kind_accuracy(pred_proto_kind, true_proto_kind, pred_id, true_id)` | conserved/sample-specific tag accuracy vs. ground truth after Hungarian alignment |
| Section-overlap rate | `section_overlap_rate(prototype_id, section_id, proto_kind)` | Self-consistency check: observed section coverage matches tagged kind (not a primary accuracy metric — circular by construction when derived from the same `prototype_id`; reported for diagnostic purposes only) |
| ARI, AMI | (see C1) | Niche recovery quality prerequisite |

**Comparators.**
- Clean-room baselines (neighborhood, PCA, diffusion): same as C1; none outputs an explicit
  conserved/private label, so each baseline's prototype assignments are post-hoc tagged by
  `section_overlap_rate` using the same threshold applied to NicheLens-ST.
- Random labeling: same as C1.

**Datasets.** From `data/cards/`:

| Card ID | Role |
|---|---|
| Synthetic instances (`n_sections >= 2`, `K_conserved >= 2`, `J_specific >= 1`) | Primary — known `proto_kind` ground truth from generator |
| `abc_atlas_zhuang_merfish` | Real, multi-section |
| `merfish_hypothalamus_moffitt` | Real, multiple subjects |
| `wang2025_matched_tma` | Real, matched multi-sample TMA |
| `xenium_lymph_node` | Real, multi-section capable |
| `xenium_breast_janesick` | Real, where data card documents ≥ 2 sections |

**Decision rule.** *Direction:* NicheLens-ST proto-kind accuracy exceeds every clean-room baseline AND
random labeling on synthetic instances with `K_conserved >= 2` and `J_specific >= 1`.
**Numeric threshold: to be fixed by pre-hoc synthetic calibration before touching real data.**
Calibration procedure: 95th percentile of random-labeling `proto_kind_accuracy` over ≥ 100 synthetic
instances with the conserved/specific mix above.

### C3 — Interpretability over generic clustering

**Hypothesis.** NicheLens-ST prototype summaries (marker enrichment, ligand-receptor interaction
ranking) provide interpretable niche characterizations that exceed a generic-clustering baseline in
marker recovery and cell-cell communication scoring.

#### Part A — Marker enrichment

| Metric | Implementation | What it locks |
|---|---|---|
| Marker recall@k | `marker_recall_at_k(pred_markers, true_markers, k=5)` | Top-5 marker recovery per prototype after Hungarian alignment |
| Macro-F1 | `macro_f1(pred_id, true_id)` | Cell assignment quality where domain annotations exist |

**Comparators.** Same clean-room baselines and random labeling as C1/C2.

**Datasets.** Synthetic benchmark (primary; known `marker_genes` per prototype from
`truth/marker_genes.json`). Real-data secondary (datasets with documented domain/cell-type
annotations from `data/cards/`):

| Card ID |
|---|
| `merfish_hypothalamus_moffitt` |
| `seqfish_mouse_embryo_lohoff` |
| `xenium_breast_janesick` |
| `abc_atlas_zhuang_merfish` |

**Decision rule.** *Direction:* NicheLens-ST marker recall@5 exceeds every clean-room baseline AND
random labeling on synthetic instances.
**Numeric threshold: to be fixed by pre-hoc synthetic calibration before touching real data.**
Calibration procedure: 95th percentile of random-labeling marker recall@5 over ≥ 100 synthetic
instances.

#### Part B — Cell-cell communication scoring

| Metric | Implementation | What it locks |
|---|---|---|
| CCC precision@k | `score_ccc_topk(pred_ranked, truth, k)["precision_at_k"]` | Fraction of top-k predictions that are planted true ligand-receptor pairs |
| CCC recall@k | `score_ccc_topk(pred_ranked, truth, k)["recall_at_k"]` | Fraction of planted positives recovered in the top-k (`hit_rate` is an alias for this key) |

**Null (decoy-only null).** A method's CCC precision@k must exceed the precision achievable by a
predictor scoring only the decoy set (elevated-but-not-colocated + random-unboosted negatives, as
defined in `docs/SYNTHETIC_BENCHMARK.md` CCC ground-truth section). The decoy-only null has an
analytically expected precision@k of 0 for k ≤ `n_ligrec_pairs`.

**Datasets.** Synthetic instances with `n_ligrec_pairs > 0` (CCC ground truth enabled via
`nichelens_st.synth.generator`). Ligand-receptor reference resources used for real-data validation:
`ligrec_omnipath` (LR-pair reference), `merfish_mouse_brain_receptor_map` (receptor mapping
reference). Real-data secondary: `merfish_hypothalamus_moffitt`, `xenium_breast_janesick`.

**Decision rule.** *Direction:* NicheLens-ST CCC precision@k exceeds the decoy-only null on synthetic
CCC instances.
**Numeric k and precision threshold: to be fixed by pre-hoc synthetic calibration before touching real
data.** Calibration procedure: set k = `n_ligrec_pairs` for each synthetic instance; determine
required precision@k at the 95th percentile of decoy-predictor scores over ≥ 50 synthetic CCC
instances.

## 2 · Multiple comparisons and reproducibility

| Control | Mechanism |
|---|---|
| Seed fixation | All synthetic instances regenerated deterministically via the `seed` parameter in `nichelens_st.synth.generator` (see `docs/SYNTHETIC_BENCHMARK.md`) |
| Result provenance | Every reported number must carry `git_sha` + `source_sha256`, per the N-T1 leaderboard machinery (`results/paper_metrics/leaderboard.json`); schema enforced by `src/nichelens_st/results_contract.py` |
| FDR control | Where multiple dataset comparisons are run simultaneously, Benjamini-Hochberg FDR correction applied at α = 0.05; rate applied to per-dataset p-values from permutation tests |
| NaN discipline | Empty inputs yield `NaN` (never a free `1.0`); single-cluster or single-cell inputs produce `NaN` and are excluded from aggregation — issue-#83 policy in `src/nichelens_st/metrics.py` |
| Determinism gate | Results contract enforced and byte-locked via `check_sync.py` before any stub is promoted to `paper_claim_ready: true` |

## 3 · Falsification criteria

| Claim | What would falsify it |
|---|---|
| C1 — Reproducible niches | NicheLens-ST ARI/AMI fails to exceed at least one clean-room baseline on the majority of synthetic instances; or `neighbor_label_agreement` does not exceed random labeling |
| C2 — Conserved/sample-specific | Proto-kind accuracy of NicheLens-ST is indistinguishable from random labeling after calibration; or the method requires post-hoc section-overlap heuristics to match baseline performance |
| C3A — Marker enrichment | Marker recall@5 of NicheLens-ST does not exceed the neighborhood clean-room baseline after Hungarian alignment on synthetic instances |
| C3B — CCC scoring | CCC precision@k of NicheLens-ST does not exceed the decoy-only null on synthetic CCC instances |

## 4 · Uncommitted thresholds (summary)

The following numeric thresholds are deliberately left open, to be fixed by pre-hoc synthetic
calibration **before any real data is processed**:

| Claim | Threshold | Calibration procedure |
|---|---|---|
| C1 — Synthetic pass gate | ARI / AMI / `neighbor_label_agreement` floor | 95th percentile of random-labeling scores over ≥ 100 synthetic instances from the full generator-parameter grid |
| C1 — Real-data pass gate | ARI / AMI floor vs. domain annotation | Same calibration transferred from synthetic grid |
| C2 — Proto-kind accuracy floor | `proto_kind_accuracy` vs. random labeling | 95th percentile of random-labeling proto-kind accuracy over ≥ 100 synthetic instances with `K_conserved >= 2`, `J_specific >= 1` |
| C3A — Marker recall floor | Marker recall@5 floor | 95th percentile of random-labeling `marker_recall_at_k` over ≥ 100 synthetic instances |
| C3B — CCC precision threshold | precision@k floor vs. decoy null | 95th percentile of decoy-predictor scores over ≥ 50 synthetic CCC instances |
| FDR — minimum dataset count | Minimum eligible datasets before FDR correction required | To be determined once dataset eligibility scan is complete |
